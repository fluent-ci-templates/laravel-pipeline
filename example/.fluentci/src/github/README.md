The following command will generate a `.github/workflows/tests.yml` file in your project:

```bash
fluentci gh init -t laravel_pipeline
```

Or, if you already have a `.fluentci` folder (generated from `fluentci init -t laravel`) in your project:

```bash
fluentci gh init
```

Generated file:

```yaml
# Do not edit this file directly. It is generated by Fluent Github Actions

name: Test
on:
  push:
    branches:
      - main
jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v2
      - uses: denoland/setup-deno@v1
        with:
          deno-version: v1.37
      - name: Setup Fluent CI CLI
        run: deno install -A -r https://cli.fluentci.io -n fluentci
      - name: Setup Dagger
        run: |
            curl -L https://dl.dagger.io/dagger/install.sh | DAGGER_VERSION=0.8.1 sh
            sudo mv bin/dagger /usr/local/bin
            dagger version
      - name: Run Dagger Pipelines
        run: dagger run fluentci laravel_pipeline

```

Feel free to edit the template generator at `.fluentci/src/github/config.ts` to your needs.
